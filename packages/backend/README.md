## Description

[Nest](https://github.com/nestjs/nest) framework TypeScript starter repository.

## Project setup

```bash
$ yarn install
```

## Compile and run the project

```bash
# development
$ yarn run start

# watch mode
$ yarn run start:dev

# production mode
$ yarn run start:prod
```

## Run tests

```bash
# unit tests
$ yarn run test

# e2e tests
$ yarn run test:e2e

# test coverage
$ yarn run test:cov
```

# FoodGuard Backend

## Prisma-NestJS-GraphQL Integration

This project uses [prisma-nestjs-graphql](https://github.com/unlight/prisma-nestjs-graphql) to automatically generate GraphQL types from the Prisma schema. This eliminates the need to manually create and maintain DTOs and entities, ensuring type consistency between the database schema and GraphQL types.

### How It Works

1. The Prisma schema in `prisma/schema.prisma` includes a generator configuration for prisma-nestjs-graphql:

```prisma
generator nestgraphql {
  provider = "prisma-nestjs-graphql"
  output   = "../src/generated"
  emitSingle = true
  emitCompiledResult = true
  addInputTypeValidation = true
  addClassValidator = true
  addClassTransformer = true
}
```

2. When you run `yarn prisma:generate`, the generator creates a single file `src/generated/index.ts` containing all the GraphQL types.

3. Import the generated types in your services and resolvers:

```typescript
import { Comment, CommentCreateInput, User } from '../generated';
```

### Benefits

- **Type Safety**: Ensures consistency between your database schema and GraphQL types
- **Reduced Boilerplate**: Eliminates the need to manually create and maintain DTOs and entities
- **Automatic Validation**: Adds class-validator decorators to input types
- **Automatic Transformation**: Adds class-transformer decorators for data transformation

### Usage in Services

When creating or updating records, use the generated input types:

```typescript
async create(content: string, authorId: string, recipeId?: string): Promise<Comment> {
  const commentData: CommentCreateInput = {
    content,
    author: {
      connect: { id: authorId },
    },
  };

  if (recipeId) {
    commentData.recipe = {
      connect: { id: recipeId },
    };
  }

  return this.prisma.comment.create({
    data: commentData,
    include: {
      author: true,
    },
  });
}
```

### Usage in Resolvers

Use the generated types for your resolver methods:

```typescript
@Resolver(() => Comment)
export class CommentsResolver {
  @Mutation(() => Comment)
  async createComment(
    @CurrentUser() user: User,
    @Args('content') content: string,
    @Args('recipeId', { nullable: true }) recipeId?: string,
  ) {
    return this.commentsService.create(content, user.id, recipeId);
  }
}
```

### Regenerating Types

After making changes to the Prisma schema, run:

```bash
yarn prisma:generate
```

This will update the generated types in `src/generated/index.ts`.
